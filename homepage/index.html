What and why?

Centage! is my five cents in defining a good fluid css framework. Every framework has its shortcomings – as has Centage! too – but it also solves some major issues in existing ones.

Reason for Centage! is purely practical. I started to redesign my own site and wanted to go for a fluid grid, mostly just for the challenge of it. At the same time I had made some tests with Less and though it seemed pretty cool, it was still hard to develop. When less.js came along, the pieces existed for putting together a next gen fluid framework.

None of the existing fluid frameworks worked as I wanted them to. Fluid 960Grid is an impressive achievement, but what if I want to start my grid from page edge? Or what about nested grids? Another wonderful undertaking is Elastic, but it's dependancy on javascript troubles me. I have my roots in print design and designing complex grid systems for newspapers. So I wanted the same grid into web as well. Especially nested, percentage width grids are a challenge that simply cannot be addressed with traditional css (afaik).

My requirements for a grid system are:

  1. It should work acceptably without javascript.
  2. It should mimic print grids as accurately as possible.
  3. Proportions of columns and gutters should be visually equal at any width.
  4. It should add minimum extra markup.
  5. No representational markup.
  6. Should work on most browsers after IE6. IE6 users can expect trouble anyway.
  7. It should be light + simple./li>

How to get there:

  1. Only use javascript and CSS3 to beautify layout, rely on robust css.
  2. Margins to columns are right. You should be able to decide not to use gutters.
  3. Take full advantage of LESS calculation capabilities to deal with subgrids.
  4. Only extra markup required are occasional ".last" -classes.
  5. Put all presentational markup into CSS/Less.
  6. Use LESS to enable more flexible, powerful and compact CSS.
  7. Only core stylesheet is necessary, extending by modularity.

Known issues / limitations:

  • Centage! is currently in alpha release state and might have bugs. It's not fully tested with IE, so you might encounter problems
  • Overriding right margin of rightmost columns requires .last -style. Any ideas how to beat this limitation are welcome (tried with negative margin container, but that messed up the percentage calculations and using last-child doesn't recognize last item on multirow grids).
  • Subpixel rendering of different browsers cause small irregularities in grid. Although a javascript-fix has been considered, irregularities are not visually very significant in real life layouts. Only browser that renders pixel-perfect is Firefox.
  • Opera works poorly supported because its horrible rendering. No decimal width percentages? Come on Opera, that's lame!

Installation

1. Download Centage! and copy to your working folder.

2. Create a new document in your preferred text editor with the extension ".less". This will be your stylesheet. For convinience set syntax to CSS.

<link rel="stylesheet/less" media="screen" href="homepage_screen.less" />

3. Add newly created file to your document head as "stylesheet/less".

4. Add less to your document head and call it.

<script src="http://lesscss.googlecode.com/files/less-1.0.35.min.js"></script>
<script type="text/javascript" charset="utf-8">
    less.env = "development";
    less.watch();
</script>

5. Put your page structure inside wrapper div (if you prefer other naming, feel free to modify centage_screen.less).

<body>
   <div id="wrap">Content here</div>
</body>

That's it, now let's move on to fun part.

Initialization

As mentioned, Centage is based on less.js, a rewrite of less. Before diving into Centage! you should familiarize yourself with less language. It's well-thought and very simple really. It'll take you only 10 minutes.

1. Open centage_sceen.less and modify settings. Set number of columns on page, default font size and line height. Let's look at it step-by-step:

2. Define main grid count. You can use other grid divisions as well, but this sets the foundation for your styles.

/*page grid columns*/
@grid:12;

3. Define default gutter width in percentages. Centage is a fluid grid, so all horizontal values are relative. Note that Centage is clever enough to remain this gutter width even in nested grids. Now isn't that something.

/*space (gutter) between columns.*/
@gut:1.5%;
@gut_int:1.5;

4. Set the page size. This sets the width of div#wrap. Width is set in ems as default to compensate text size changes, but feel free to use your preferred units.

/*page size*/
@max-width:110em;
@min-width:66em;

5. Set font and line height. The default lineheight also defines the baseline grid.

/*typography, baseline grid*/
@font:Arial, sans-serif;

@bodyfont:75%; /*set the body font size in percentages*/
@fontpx:12; /*standard font size in pixels, needed for calculations*/
@fontpxem:12em; /*this is needed for calculations*/

/*standard line height */
@line:1.5em;

@linepx:18; /*standard line height in pixels, please. Calculate it like this: @fontpx * @line. */

Defining styles

Setting up columns

The columns are easy to set up using two "functions" in less: .col() and .sub-col()

.col([column count], [vertical padding]*, [horizontal padding]*)

/* = 3 columns wide, one line vertical padding, horizontal padding as wide as gutter */ 
#sidebar {
	.col(3, @line, @gut);} 


/* Nested column, 4 columns inside 8 column container. Paddings can include calculations. */
.section {
	.sub-col(8,4,@line, 2*@gut);}
